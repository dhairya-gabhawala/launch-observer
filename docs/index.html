How it works

Launch Observer is designed for quick adoption. Set up a session, capture requests, and drill down in seconds.

Step 1

Name your session

Start a session with a site name and optionally lock to a browser tab.

Step 2

Choose domains

Enable the domains and services you want to track. Add custom domains if needed.

Step 3

Validate with UAT

Import your UAT assertion JSON for the site and enable validations when starting a session.

Step 4

Inspect the report

Browse requests, open details, and decode payloads with context markers for event types.

UAT assertions, explained

UAT assertions are JSON rules that verify your analytics data. Each assertion includes one or more conditions and optional page-level count checks, plus optional global gates to include or exclude services.

Per-site configs

Upload one JSON file per site. Assertions only apply to sessions created for that site.

Request + page scope

Validate a single request or enforce page-level rules like “pageview fires exactly once.”

Shareable reports

Export a PDF of pass/fail results for teams and stakeholders.

Schema notes

The in-app Help panel documents required vs optional fields, operator types, global service gates, and examples.
